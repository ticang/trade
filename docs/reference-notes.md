## 2. 核心经验

### 2.1 平台参考

- 提到"聚宽看看呗"——参考 **聚宽（JoinQuant）**，A 股主流量化平台（行情数据 + 回测）。

### 2.2 核心系统：multi-agent 因子挖掘

> 原文：「策略方案为手搓了个 multiagents 因子挖掘系统，自动挖掘因子，挖掘因子能在策略里发现相对大概率上涨的股票，再自动迭代挖掘生成新因子」

要点：

- multi-agent 架构自动挖掘 alpha 因子
- 因子用于在策略中选出"相对大概率上涨"的标的
- 自动迭代生成新因子（形成闭环）

### 2.3 差异化 alpha：散户情绪反向

聊天记录还原要点：

- "现在真发现散户的情绪指标"
- "散户就是对手盘，注定被割韭菜"
- "反方向思考容错率会大很多"
- "情绪被作为因子参与"（策略）
- "昨晚情绪高，显示情绪有一致性"

结论：**散户情绪作为反向因子**参与策略；情绪具有一致性，可量化、可复用。

### 2.4 情绪数据来源

聊天记录还原要点：

- "我拉各种群收集数据"
- "抖音博主小群、小红书群"
- "抖音热门博主评论区"

结论：从社媒（抖音 / 小红书）的私域群聊与评论区采集散户情绪数据。

### 2.5 特殊节点行情定制策略

聊天记录还原要点：

- "针对行情特别节点行情定制策略，结合散户情绪"
- "定制策略虽然耗时间、可用性短"
- "但是带来的收益率是爆发式的"

结论：针对特殊行情节点定制策略 + 结合情绪；这类策略生命周期短，但收益爆发式，需要快速迭代。

### 2.6 可视化思路（行情分析图）

行情分析截图展示的分析范式：

- **三合一图表**：价格走势 + 成交量 + 散户情绪曲线
- 散户情绪全天时间轴
- 关键时点信号标注（起飞 / 抄底 / 清仓 / 防套 等）

## 3. 提炼要点

1. **multi-agent 自动因子挖掘** 是技术核心（自动产出因子 + 迭代）。
2. **散户情绪反向** 是差异化 alpha 来源（散户 = 对手盘）。
3. **情绪数据多渠道采集**（含社媒私域群聊 / 评论区）。
4. **特殊节点定制策略** 收益爆发式，但需快速迭代、生命周期短。
5. **价格 + 成交量 + 情绪 三合一** 是分析范式。

## 5. 合规提示（原资料未明确，本设计补充）

- 私域群聊（抖音 / 小红书群）与评论区爬取涉及**平台 ToS** 与《个人信息保护法》(PIPL)。
- 本设计将社媒私域采集设为**二期**，默认关闭，需用户显式知情开启。
- 一期仅使用**公开数据**（东方财富股吧 / 雪球 / 微博公开 UGC + 涨跌停家数 / 连板高度 / 封板率 / 北向资金 / 融资融券等市场情绪指标）。
