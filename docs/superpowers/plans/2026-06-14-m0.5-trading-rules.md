# M0.5 交易规则子模块 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development. Checkbox tracking. 实现与审查分离。

**Goal:** 在 M0 的 `trading_rule` 表与 TradingRuleProvider 基础上，录入 A 股交易规则 v1 数据集（沪深主板/创业板/科创板/北交所/ETF/可转债，2020 至今），建立 source_confidence 体系（verified/provisional，provisional 阻断实盘），提供 golden cases 验证区间查询正确性。覆盖设计 §4.1.3 + §11 M0.5。

**Architecture:** 规则数据以 YAML 种子集形式版本化（`quant/providers/data/rules_v1.yaml`），loader 读入并写入 `trading_rule` 表（复用 M0 的 check_no_overlap 应用层校验保证区间不重叠），TradingRuleProvider.rules_for 返回真实规则；source_confidence 标注每条规则可信度，`rules_for(..., require_verified=True)` 在含 provisional 时拒绝（阻断实盘）。

**Tech Stack:** Python 3.11+ · PyYAML · pytest（复用 M0 基建，无新依赖）

**关联：** 设计 v0.5 §4.1.3（基础数据与交易规则）、§4.1.6（规则变更需人工 golden cases）、§6（trading_rule 表）、§11 M0.5

**前置：** M0 完成（trading_rule 表 + TradingRuleProvider + check_no_overlap + classify_symbol 在 main）。本计划在 `feat/m0.5-trading-rules` 分支。

**验收标准（设计 §11 M0.5）：**
1. fixture 100% 通过（golden cases 全绿）
2. 区间查询无重叠命中（check_no_overlap 对种子数据返回空）
3. 过户费等无权威项标 provisional 且阻断实盘（rules_for require_verified=True 时含 provisional 返回拒绝）
4. rules_for 返回真实规则（不再是空表 None）

## 数据准确度策略（核心）

**verified（高置信，结构性规则，可作回测/实盘依据）：**
- 价格最小变动单位（tick）：主板/创业板/科创板/北交所股票 0.01；可转债 0.001
- 涨跌幅：主板 ±10%（ST/*ST ±5%、退市整理 ±10%）；创业板/科创板 ±20%（2020-08-24 创业板改革后 / 科创板开板即）；北交所 ±30%
- 结算 T+N：股票 T+1；可转债 T+0；跨境 ETF T+0，普通 ETF T+1
- 数量：主板/创业板 100 整数倍；科创板 200 起 1 递增；北交所 100 起 1 递增

**provisional（待官方核验，阻断实盘）：**
- 费用：印花税 0.05%（卖出，2023-08-28 减半）、过户费、佣金、经手费 —— 费率随政策/券商变动，需官方来源核验后升 verified
- 这些标注 provisional，rules_for(..., require_verified=True) 会拒绝
