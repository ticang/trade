# M1.5 策略引擎 + 组合优化器 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development. Checkbox tracking. 实现与审查分离。**跨多个上下文窗口；每任务边界更新 memory。**

**Goal:** 在 M1 因子/回测/风控之上，交付策略引擎：Strategy/on_fill 接口、多策略调度+隔离、组合优化器（cvxpy QP 标量化+主板 100 股整数化+gap 报告）、止损止盈/策略生命周期/再平衡。覆盖设计 §4.4 + §11 M1.5 + v0.5-scope。

**Architecture:** 接口先行——Strategy/StrategyRunner/PortfolioOptimizer 均为 Protocol/抽象。策略输出 Signal（含 stop_loss/take_profit/trailing），StrategyRunner 装配 BarContext（因子面板+规则+持仓），多策略并行+冲突裁决+目标持仓差分下单。组合优化器标量化目标 `max α'w − λ·TE² − γ·turnover`，QP 连续解→round+约束修复启发式→gap 报告。复用 M1 BacktestEngine/RiskEngine/SimBroker/FactorRegistry。

**Tech Stack:** Python 3.11+ · cvxpy(+OSQP) · 复用 M0/M0.5/M1 全栈

**关联：** 设计 v0.5 §4.4（策略引擎）、§4.4.3（组合优化）、§4.4.4（止损/生命周期/再平衡）、§11 M1.5

**前置：** M0+M0.5+M1 完成（main）。本计划在 `feat/m1.5-strategy`。

**验收标准（设计 §11 M1.5）：**
1. 多策略并行回测（隔离：单策略异常不阻断整 bar）
2. 优化约束满足（单票/行业/换手上限）
3. QP-vs-整数化 gap < 阈值
4. 换手 ≤ 配置上限
5. 生命周期状态机可迁移（draft→...→offline）

## Task 3: 组合优化器（cvxpy QP + 主板 100 股整数化 + gap）
**Files:** `quant/strategy/optimizer.py`, `tests/quant/test_strategy_optimizer.py`
- [ ] **先装 cvxpy**：`.venv/bin/pip install --index-url https://pypi.org/simple cvxpy`，pyproject 加 `cvxpy>=1.4`。
- [ ] PortfolioOptimizer.optimize(signals, current_weights, constraints, benchmark, λ, γ)：标量化目标 `max α'w − λ·TE² − γ·turnover`（α=signal.strength，TE=相对基准跟踪误差，turnover=|w-current|）。约束：单票上限/行业暴露/换手上限（惩罚为主+硬上限）。QP 连续解 → 主板 100 股整数化 round（按 alpha 降序贪心填充，违反约束降 lot 重试）→ 返回 (integer_weights, gap)。多 lot/MIQP 留后续扩展。
- [ ] gap = |objective(integer) - objective(continuous)|；gap>阈值告警（返回 flag）。
- [ ] 测试：连续解满足约束；整数化后约束满足；gap 计算正确；约束不可行时回退连续解。
- [ ] commit `add portfolio optimizer with qP and multi-lot integerization`
